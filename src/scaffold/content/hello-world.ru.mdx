## Что такое CitePo?

CitePo — это легковесная платформа для публикации блогов. Вы пишете посты в **MDX** (Markdown + JSX), отправляете в Git, и ваш сайт автоматически собирается и разворачивается.

<Callout type="tip">
  MDX позволяет использовать богатые компоненты внутри Markdown. Этот пост сам является MDX-файлом — прокрутите до раздела "MDX-компоненты", чтобы увидеть, что доступно!
</Callout>

## Написание вашего первого настоящего поста

Создайте новый файл `.mdx` в директории `content/`. Каждый пост требует **frontmatter** в начале:

```mdx
---
title: Мой Потрясающий Пост
description: Краткое описание поста
date: 2025-01-27
tags: [tech, tutorial]
---

Ваш markdown-контент здесь...
```

### Поддерживаемые поля Frontmatter

| Поле          | Обязательно | Описание                              |
| ------------- | ----------- | ------------------------------------- |
| `title`       | Да          | Заголовок поста                       |
| `description` | Да          | Краткое описание (для SEO)            |
| `date`        | Да          | Дата публикации (YYYY-MM-DD)          |
| `tags`        | Нет         | Массив тегов для категоризации        |
| `authors`     | Нет         | Массив имен авторов                   |
| `coverImage`  | Нет         | Путь к обложке в `asset/`             |
| `draft`       | Нет         | `true` чтобы скрыть в продакшене      |

## Примеры форматирования

### Стили текста

Вы можете писать **жирным**, _курсивом_, ~~зачеркнутым~~ и `встроенным кодом`.

### Блоки кода

```typescript
function приветствовать(имя: string): string {
  return `Привет, ${имя}! Добро пожаловать в CitePo.`
}

console.log(приветствовать('Мир'))
```

### Списки

- Пишите в вашем любимом редакторе (VS Code, Cursor, Vim...)
- Просматривайте с `npx citepo dev`
- Собирайте с `npx citepo build`
- Разворачивайте через push в Git
